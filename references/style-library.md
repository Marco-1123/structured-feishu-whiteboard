# Style Library

第一版内置 3 套精选风格。它们是“UI 设计 token”，不是固定模板截图。生成时把风格规则应用到 5 个版式之一。

## 通用原则

- 信息密度高但不拥挤。
- 使用清晰网格、卡片、分隔线和标签。
- 不使用渐变、透明、模糊、复杂插画。
- 强调色只用于关键结论、推荐项、阶段编号或状态标签。
- 字号层级建议：
  - 标题：44 到 64。
  - 主结论：28 到 38。
  - 卡片标题：24 到 30。
  - 正文：18 到 22。
  - 标签：16 到 18。

## 1. 专业蓝白

定位：

- 默认风格。
- 适合大多数方案、报告、产品规划、业务分析。
- 清爽、理性、接近现代 SaaS / 咨询汇报。

颜色：

- canvas: `#F7FAFC`
- surface: `#FFFFFF`
- surface-muted: `#EEF4FA`
- ink: `#172033`
- secondary: `#5C6B82`
- border: `#D8E2EF`
- accent: `#2563EB`
- accent-soft: `#DBEAFE`
- success: `#0F766E`

样式：

- 卡片圆角 `rx=10` 到 `14`。
- 细边框 1.5 到 2px。
- 标题区可使用蓝色竖条、顶部细线或小标签。
- 推荐项使用蓝色边框或浅蓝底，不使用大面积强蓝。

适配版式：

- 结论先行、对比矩阵、路线图优先。

## 2. 深色强调

定位：

- 适合高管汇报、战略判断、关键决策、强结论页面。
- 深色底或深色主面板，形成强焦点。

颜色：

- canvas: `#101828`
- surface: `#182236`
- surface-muted: `#22314A`
- ink: `#F8FAFC`
- secondary: `#B7C2D5`
- border: `#33445F`
- accent: `#38BDF8`
- accent-2: `#A3E635`
- warning: `#FACC15`

样式：

- 背景深，卡片分层清楚。
- 用青蓝作为主强调，黄绿只用于少量关键数字或推荐标识。
- 卡片圆角 `rx=12` 到 `18`。
- 边框可以比蓝白风更弱，但必须保持可见。
- 小字不要放在高饱和色块上。

适配版式：

- 结论先行、问题拆解、流程/价值链优先。

## 3. 暖灰编辑

定位：

- 适合研究总结、观点提炼、知识型报告、偏 editorial 的方案展示。
- 比蓝白风更柔和，但仍保持专业。

颜色：

- canvas: `#F3EFE7`
- surface: `#FFFDF8`
- surface-muted: `#E8E1D6`
- ink: `#22201C`
- secondary: `#6E675D`
- border: `#D6CABC`
- accent: `#B45309`
- accent-soft: `#F7D9B4`
- olive: `#6B7D3A`

样式：

- 卡片圆角 `rx=8` 到 `12`。
- 分隔线、编号和标签使用暖棕或橄榄色。
- 适合留白更大的画板，不要做过密 dashboard。
- 标题可更大、更像报告页标题。

适配版式：

- 问题拆解、结论先行、对比矩阵优先。

## 风格选择

- 用户没有偏好：默认专业蓝白。
- 用户要“高级、决策、战略、强冲击”：选深色强调。
- 用户要“温和、知识感、编辑感、研究感”：选暖灰编辑。
- 用户提到特定 UI 审美时，先判断它更接近以上哪套，再用最接近的一套表达；不要临时创造第四套风格。
