# Style Library

V3 内置 6 套基础风格；V3.1 新增 Apple Studio 和 Linear Command 两套生产候选风格。V3.5 增加 6 套标杆启发风格：4 套工作型增强风格，2 套用户主动选择的创意张力风格。

这些风格是“UI 设计 token + 组件语法”，不是固定模板截图。生成时把风格规则应用到脚本化版式之一。V3.5 的重点不是增加随机皮肤，而是让不同材料进入更匹配的信息表达语言，同时保持结构化、可读性和排版秩序。

## 通用原则

- 信息密度高但不拥挤。
- 使用清晰网格、卡片、分隔线和标签。
- 不使用渐变、透明、模糊、复杂插画。
- 强调色只用于关键结论、推荐项、阶段编号或状态标签。
- 颜色必须有语义：主色表达主线，风险色只表达风险，成功色只表达正向结果；同一组并列信息不能随机换色。
- 创意风格只能由用户主动选择，不能作为默认自动风格；创意风格也必须通过 JSON brief、确定性渲染器和布局检查。
- 字号层级建议：
  - 标题：44 到 64。
  - 主结论：28 到 38。
  - 卡片标题：24 到 30。
  - 正文：18 到 22。
  - 标签：16 到 18。

## 1. 专业蓝白

定位：

- 默认风格。
- 适合大多数方案、报告、产品规划、业务分析。
- 清爽、理性、接近现代 SaaS / 咨询汇报。

颜色：

- canvas: `#F7FAFC`
- surface: `#FFFFFF`
- surface-muted: `#EEF4FA`
- ink: `#172033`
- secondary: `#5C6B82`
- border: `#D8E2EF`
- accent: `#2563EB`
- accent-soft: `#DBEAFE`
- success: `#0F766E`

样式：

- 卡片圆角 `rx=10` 到 `14`。
- 细边框 1.5 到 2px。
- 标题区可使用蓝色竖条、顶部细线或小标签。
- 推荐项使用蓝色边框或浅蓝底，不使用大面积强蓝。

适配版式：

- 结论先行、对比矩阵、路线图优先。

## 风格选择

- 用户没有偏好：默认专业蓝白。
- 用户要“高级、决策、战略、强冲击”：选深色强调。
- 用户要“温和、知识感、编辑感、研究感”：选暖灰编辑。
- 用户要“更像飞书、日常工作台、清爽但不只有蓝白”：选飞书浅灰。
- 用户要“流程、状态、风险、指标、闭环”：选飞书状态。
- 用户要“高管决策、深色强调但仍可读、不像海报”：选飞书决策深色。
- 用户要“更高级、克制、留白更舒服、Apple 感”：选 Apple Studio 或 Apple Report。
- 用户要“系统感、策略感、工程/产品决策感、Linear 感”：选 Linear Command 或 Linear System。
- 用户要“数据化、经营复盘、指标趋势、Dashboard 感”：选 Stripe Data。
- 用户要“规则、验收、技术方案、发布检查、高对比”：选 Vercel Precision。
- 用户明确要“更大胆、有设计感、发布感、方法论包装”：可选 Neo Grid Bold。
- 用户明确要“强视觉、蓝图、宣讲页、双色海报”：可选 Riptide Cobalt。
- 用户提到特定 UI 审美时，先判断它更接近以上哪套，再用最接近的一套表达；不要临时创造第七套风格。

## 7. Apple Studio

定位：

- 高级、干净、留白充足的工作汇报页。
- 适合产品概念、能力介绍、策略总览和高质量 onepage。
- 当前作为 V3.1 生产候选风格，必须使用专属 Apple 版式函数，不允许只套通用模板换色。
- 新增或修改 Apple 样例后，必须导出飞书侧预览，检查底部边距、卡片内边距、箭头连接和区域留白。

颜色：

- canvas: `#F5F5F7`
- surface: `#FFFFFF`
- surface-muted: `#F2F2F7`
- ink: `#1D1D1F`
- secondary: `#6E6E73`
- border: `#D2D2D7`
- accent: `#007AFF`
- accent-soft: `#E8F2FF`
- success: `#248A3D`

样式：

- 通过宽松留白、强标题层级和柔和分组制造质感。
- 少用粗边框和密集分隔线，更多依赖间距组织信息。
- 胶囊标签优先于硬边框标签。
- 不使用真实模糊、透明和玻璃效果，避免飞书 SVG 兼容问题。

适配版式：

- 已覆盖：结论先行、路线图/阶段规划、流程/价值链、对比矩阵。
- 结论先行、路线图/阶段规划、流程/价值链必须走 Apple 专属布局，保持顶部标题区、结论卡、主体卡片和底部结论区的统一边距节奏。

## 8. Linear Command

定位：

- 克制、精准、系统化的产品/工程/策略表达。
- 适合能力架构、决策路径、工程流程和团队协作机制。
- 当前用户反馈：整体符合预期，可作为 V3.1 重点生产候选。

颜色：

- canvas: `#F7F8FA`
- surface: `#FFFFFF`
- surface-muted: `#F1F3F6`
- ink: `#111827`
- secondary: `#5F6B7A`
- border: `#D8DEE8`
- accent: `#5E6AD2`
- accent-soft: `#ECEEFE`
- success: `#16A085`

样式：

- 减少装饰，强调精准对齐、清晰分区和低噪声。
- 深色只用于轨道、局部强调或决策条，不使用大面积深色底。
- 飞书预览可能弱化深底白字，因此正文默认仍放在浅色面板上。

适配版式：

- 已覆盖：结论先行、路线图/阶段规划、流程/价值链、对比矩阵。
- 路线图和流程/价值链必须检查箭头是否从前一卡片边缘连接到后一卡片边缘，不允许只有漂浮箭头头部。
- onepage 大画布可使用该风格的通用组件语法，但发布前仍需要单独人工检查深色轨道、箭头和区域留白。

- 适合产品介绍、能力总览、策略稿和高质量一页汇报。

颜色：

- canvas: `#F5F5F7`
- surface: `#FFFFFF`
- surface-muted: `#F2F2F7`
- ink: `#1D1D1F`
- secondary: `#6E6E73`
- border: `#D2D2D7`
- accent: `#007AFF`
- accent-soft: `#E8F2FF`
- success: `#248A3D`

规则：

- 不适合超高密度经营看板。

## 13. Neo Grid Bold

定位：

- V3.5 创意张力风格，用户主动选择，不作为默认自动风格。
- 借鉴编辑海报、设计周物料和 Neo-Grid Bold 的强网格感。
- 适合版本发布、方法论介绍、愿景、概念包装和高记忆点观点页。

颜色：

- canvas: `#F5F4EF`
- surface: `#F5F4EF`
- surface-muted: `#ECECE8`
- ink: `#0A0A0A`
- secondary: `#5F5F59`
- border: `#0A0A0A`
- accent: `#E6FF3D`

规则：

- 只允许一个强强调色：electric lemon。
- 卡片和分区使用直角、重边框和固定网格。

- 即使视觉大胆，文本仍必须短句化、分区清晰、不得重叠。

- 适合概念蓝图、宣讲页、战略主张、视觉冲击较强的一页图。

颜色：

- canvas: `#FDF0E0`
- surface: `#FFFFFF`
- surface-muted: `#FDF0E0`
- ink: `#1A2240`
- secondary: `#4E5873`
- border: `#1A2240`
- accent: `#375DFE`
- accent-soft: `#E8ECFF`

规则：

- 使用钴蓝 + 暖奶油双色，不引入第三个装饰色。
- 小字号正文必须放在白色或浅色内容区，不能直接压在强钴蓝色块上。
- 适合高冲击表达，但不能牺牲结构、对齐和可读性。
