# Style Library

V3 内置 6 套基础风格；V3.1 新增 Apple Studio 和 Linear Command 两套生产候选风格。它们是“UI 设计 token + 组件语法”，不是固定模板截图。生成时把风格规则应用到脚本化版式之一。

## 通用原则

- 信息密度高但不拥挤。
- 使用清晰网格、卡片、分隔线和标签。
- 不使用渐变、透明、模糊、复杂插画。
- 强调色只用于关键结论、推荐项、阶段编号或状态标签。
- 字号层级建议：
  - 标题：44 到 64。
  - 主结论：28 到 38。
  - 卡片标题：24 到 30。
  - 正文：18 到 22。
  - 标签：16 到 18。

## 1. 专业蓝白

定位：

- 默认风格。
- 适合大多数方案、报告、产品规划、业务分析。
- 清爽、理性、接近现代 SaaS / 咨询汇报。

颜色：

- canvas: `#F7FAFC`
- surface: `#FFFFFF`
- surface-muted: `#EEF4FA`
- ink: `#172033`
- secondary: `#5C6B82`
- border: `#D8E2EF`
- accent: `#2563EB`
- accent-soft: `#DBEAFE`
- success: `#0F766E`

样式：

- 卡片圆角 `rx=10` 到 `14`。
- 细边框 1.5 到 2px。
- 标题区可使用蓝色竖条、顶部细线或小标签。
- 推荐项使用蓝色边框或浅蓝底，不使用大面积强蓝。

适配版式：

- 结论先行、对比矩阵、路线图优先。

## 2. 深色强调

定位：

- 适合高管汇报、战略判断、关键决策、强结论页面。
- 深色底或深色主面板，形成强焦点。

颜色：

- canvas: `#101828`
- surface: `#182236`
- surface-muted: `#22314A`
- ink: `#F8FAFC`
- secondary: `#B7C2D5`
- border: `#33445F`
- accent: `#38BDF8`
- accent-2: `#A3E635`
- warning: `#FACC15`

样式：

- 背景深，卡片分层清楚。
- 用青蓝作为主强调，黄绿只用于少量关键数字或推荐标识。
- 卡片圆角 `rx=12` 到 `18`。
- 边框可以比蓝白风更弱，但必须保持可见。
- 小字不要放在高饱和色块上。

适配版式：

- 结论先行、问题拆解、流程/价值链优先。

## 3. 暖灰编辑

定位：

- 适合研究总结、观点提炼、知识型报告、偏 editorial 的方案展示。
- 比蓝白风更柔和，但仍保持专业。

颜色：

- canvas: `#F3EFE7`
- surface: `#FFFDF8`
- surface-muted: `#E8E1D6`
- ink: `#22201C`
- secondary: `#6E675D`
- border: `#D6CABC`
- accent: `#B45309`
- accent-soft: `#F7D9B4`
- olive: `#6B7D3A`

样式：

- 卡片圆角 `rx=8` 到 `12`。
- 分隔线、编号和标签使用暖棕或橄榄色。
- 适合留白更大的画板，不要做过密 dashboard。
- 标题可更大、更像报告页标题。

适配版式：

- 问题拆解、结论先行、对比矩阵优先。

## 风格选择

- 用户没有偏好：默认专业蓝白。
- 用户要“高级、决策、战略、强冲击”：选深色强调。
- 用户要“温和、知识感、编辑感、研究感”：选暖灰编辑。
- 用户要“更像飞书、日常工作台、清爽但不只有蓝白”：选飞书浅灰。
- 用户要“流程、状态、风险、指标、闭环”：选飞书状态。
- 用户要“高管决策、深色强调但仍可读、不像海报”：选飞书决策深色。
- 用户要“更高级、克制、留白更舒服、Apple 感”：选 Apple Studio。
- 用户要“系统感、策略感、工程/产品决策感、Linear 感”：选 Linear Command。
- 用户提到特定 UI 审美时，先判断它更接近以上哪套，再用最接近的一套表达；不要临时创造第七套风格。

## 4. 飞书浅灰

定位：

- 日常业务汇报、项目同步、团队工作台。
- 比专业蓝白更接近飞书工作台界面，强调中性色、浅边框和克制蓝色。

颜色：

- canvas: `#F5F7FA`
- surface: `#FFFFFF`
- surface-muted: `#F0F3F8`
- ink: `#1F2329`
- secondary: `#646A73`
- border: `#DEE4ED`
- accent: `#3370FF`
- accent-soft: `#EAF0FF`
- success: `#2B8F77`

样式：

- 以浅灰页面和白色卡片为主。
- 蓝色只用于编号、主结论和关键路径。
- 卡片边框轻，避免大面积色块。

适配版式：

- 结论先行、路线图、对比矩阵优先。

## 5. 飞书状态

定位：

- 流程、状态、风险、指标和闭环说明。
- 用低饱和功能色表达状态，不做彩虹式分类。

颜色：

- canvas: `#F6F8FB`
- surface: `#FFFFFF`
- surface-muted: `#EEF3F7`
- ink: `#1F2329`
- secondary: `#646A73`
- border: `#DDE6EE`
- accent: `#3370FF`
- accent-soft: `#E8F3FF`
- success: `#2B8F77`
- warning: `#C98100`

样式：

- 蓝色表达主流程。
- 绿色表达完成、复用或正向输出。
- 橙色只用于风险和待确认，不作为大面积背景。

适配版式：

- 流程/价值链、路线图、问题拆解优先。

## 6. 飞书决策深色

定位：

- 高管汇报、方案选择、关键判断。
- 深色强调但不使用大面积深底，避免飞书侧文本颜色解析不稳定导致可读性下降。

颜色：

- canvas: `#F2F5FA`
- surface: `#FFFFFF`
- surface-muted: `#E9EEF7`
- ink: `#1F2329`
- secondary: `#4E5969`
- border: `#CBD5E1`
- accent: `#1F3A5F`
- accent-soft: `#DCE8F6`
- success: `#2B8F77`

样式：

- 深色只用于标题强调、推荐项边界、关键判断线和少量标签。
- 表格、卡片和结论区使用浅底，确保飞书侧预览和编辑态都可读。
- 不使用高饱和多色，只保留深蓝和少量绿色。

适配版式：

- 对比矩阵、结论先行、问题拆解优先。

## 7. Apple Studio

定位：

- 高级、干净、留白充足的工作汇报页。
- 适合产品概念、能力介绍、策略总览和高质量 onepage。
- 当前作为 V3.1 生产候选风格，必须使用专属 Apple 版式函数，不允许只套通用模板换色。
- 新增或修改 Apple 样例后，必须导出飞书侧预览，检查底部边距、卡片内边距、箭头连接和区域留白。

颜色：

- canvas: `#F5F5F7`
- surface: `#FFFFFF`
- surface-muted: `#F2F2F7`
- ink: `#1D1D1F`
- secondary: `#6E6E73`
- border: `#D2D2D7`
- accent: `#007AFF`
- accent-soft: `#E8F2FF`
- success: `#248A3D`

样式：

- 通过宽松留白、强标题层级和柔和分组制造质感。
- 少用粗边框和密集分隔线，更多依赖间距组织信息。
- 胶囊标签优先于硬边框标签。
- 不使用真实模糊、透明和玻璃效果，避免飞书 SVG 兼容问题。

适配版式：

- 已覆盖：结论先行、路线图/阶段规划、流程/价值链、对比矩阵。
- 结论先行、路线图/阶段规划、流程/价值链必须走 Apple 专属布局，保持顶部标题区、结论卡、主体卡片和底部结论区的统一边距节奏。

## 8. Linear Command

定位：

- 克制、精准、系统化的产品/工程/策略表达。
- 适合能力架构、决策路径、工程流程和团队协作机制。
- 当前用户反馈：整体符合预期，可作为 V3.1 重点生产候选。

颜色：

- canvas: `#F7F8FA`
- surface: `#FFFFFF`
- surface-muted: `#F1F3F6`
- ink: `#111827`
- secondary: `#5F6B7A`
- border: `#D8DEE8`
- accent: `#5E6AD2`
- accent-soft: `#ECEEFE`
- success: `#16A085`

样式：

- 减少装饰，强调精准对齐、清晰分区和低噪声。
- 深色只用于轨道、局部强调或决策条，不使用大面积深色底。
- 飞书预览可能弱化深底白字，因此正文默认仍放在浅色面板上。

适配版式：

- 已覆盖：结论先行、路线图/阶段规划、流程/价值链、对比矩阵。
- 路线图和流程/价值链必须检查箭头是否从前一卡片边缘连接到后一卡片边缘，不允许只有漂浮箭头头部。
- onepage 大画布可使用该风格的通用组件语法，但发布前仍需要单独人工检查深色轨道、箭头和区域留白。
