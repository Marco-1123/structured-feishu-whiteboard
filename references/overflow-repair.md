# Overflow Repair

当 SVG 检查或人工预览发现文字出框、堆叠、裁切、拥挤或乱码时，按本顺序修复。不要一上来缩小字号。

## 修复顺序

1. **清理文本**：删除异常符号、重复句、长 URL、脚注和过程性信息。
2. **压缩内容**：把长句改成短句，把 4 条以上要点合并成 2 到 3 条。
3. **手动换行**：用 `<tspan>` 明确控制每行文本，避免浏览器和飞书重排差异。
4. **调整容器**：增加卡片高度、加宽列、增加模块间距。
5. **扩展同页区域**：内容本身合理但空间不足时，短内容可增高画布，长文优先扩大 onepage 中的对应区域。
6. **拆成多图**：如果调整后仍需要低于字号底线，必须拆成多张画板。

## 禁止修复方式

- 不要把正文缩到 18 以下。
- 不要把标签缩到 16 以下。
- 不要让正文贴近卡片边框。
- 不要把多段长文塞进一个 `<text>`。
- 不要用 opacity、filter、clipPath、mask 或图片化文字绕过问题。
- 不要把所有内容做成一张大截图。

## 常见问题

| 问题 | 判断 | 处理 |
|-|-|-|
| 文字出框 | 文本超过卡片宽度或高度 | 压缩文本、换行、增高卡片 |
| 内容堆叠 | 多个节点坐标过近或行距不足 | 增加行距、模块间距、画布高度 |
| 画面拥挤 | 视觉上能读但没有呼吸感 | 减少模块或拆图，不缩字号 |
| 乱码 | 特殊符号、控制字符、未转义文本进入 SVG | 清理输入并转义文本字符 |
| 固定比例硬塞 | 画布或区域没有随内容扩展 | 短内容增高画布；长文扩大同页区域 |

## 最小安全距离

- 卡片内边距：至少 32。
- 标题到正文：至少 28。
- 正文行距：字号的 1.45 到 1.6 倍。
- 标签到底边：至少 28。
- 箭头到文字：至少 20。
- 画布边缘安全距：至少 64。

## 复查

每次修复后重新运行：

```bash
npx -y @larksuite/whiteboard-cli@^0.2.11 -i diagram.svg -o diagram.png -f svg
npx -y @larksuite/whiteboard-cli@^0.2.11 -i diagram.svg -f svg --check
```

然后人工查看 PNG。工具通过只能证明没有明显重叠，不能证明画面舒适。
