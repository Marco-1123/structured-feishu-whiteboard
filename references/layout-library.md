# Layout Library

只从下面 5 个版式中选择。可以根据内容数量调整列数、卡片宽度、间距和高度，但不要改变版式的核心表达逻辑。

## 1. 结论先行

适用：

- 方案汇报、研究总结、战略建议、执行建议。
- 材料有明确主张或推荐路径。

结构：

- 顶部：主题标题 + 一句话主结论。
- 中部：3 到 5 个支撑模块卡片。
- 底部：关键行动、风险或下一步。

排版规则：

- 主结论占画板最强视觉位置。
- 支撑模块用等宽卡片，标题短、正文少。
- 行动项用窄条或底部栏，不抢主结论。

失败条件：

- 没有明确主结论。
- 内容主要是时间顺序或流程机制。

## 2. 问题拆解

适用：

- 诊断报告、现状分析、增长问题、组织问题、产品问题。
- 材料包含问题、原因、影响、解决方向。

结构：

- 左侧或顶部：核心问题。
- 中部：3 到 5 个原因或维度。
- 右侧或底部：对应策略、机会或治理动作。

排版规则：

- 使用“问题 -> 原因 -> 方案”的阅读方向。
- 原因模块之间保持同级，不混入行动项。
- 每个原因最好对应一个影响或证据。

失败条件：

- 材料不是诊断型，而是计划型或流程型。

## 3. 对比矩阵

适用：

- 竞品分析、方案比较、路径选择、能力评估。
- 材料包含多个对象和多个评价维度。

结构：

- 行：对象、方案、产品、阶段或角色。
- 列：评价维度。
- 右侧或底部：推荐结论。

排版规则：

- 维度控制在 3 到 5 个。
- 对象控制在 3 到 6 个。
- 单元格只放短语或符号化判断，不放长段落。
- 推荐项用强调色、边框或徽标突出。

失败条件：

- 只有单一对象，没有可比维度。
- 每个单元格都需要长解释。

## 4. 路线图 / 阶段规划

适用：

- 项目计划、产品路线、能力建设、实施路径。
- 材料天然按时间、阶段或成熟度展开。

结构：

- 横向阶段轴：3 到 5 个阶段。
- 每个阶段：目标、关键动作、交付物或指标。
- 末端：目标状态或成功标准。

排版规则：

- 阶段之间用箭头或连接线表达推进。
- 每阶段信息量保持均衡。
- 时间标签可以简短放在阶段标题附近。

失败条件：

- 没有顺序关系。
- 内容是多对象比较而非推进计划。

## 5. 流程 / 价值链

适用：

- 业务流程、用户旅程、系统链路、价值创造过程。
- 材料包含输入、处理、输出、角色或环节。

结构：

- 左到右主流程：4 到 7 个节点。
- 每个节点：动作、责任方、输入输出或关键风险。
- 上下辅助层：支撑能力、数据、约束或指标。

排版规则：

- 节点之间必须有明确方向。
- 连接线少而清晰，避免网状混乱。
- 支撑层用细卡片或横条，不打断主流程。

失败条件：

- 材料主要是结论陈述，没有过程关系。
- 节点超过 7 个且不能合并。

## 选择优先级

1. 有明确推荐或判断：结论先行。
2. 有问题和原因：问题拆解。
3. 有对象和维度：对比矩阵。
4. 有阶段和时间：路线图。
5. 有输入输出和环节：流程/价值链。

如果多个版式都可用，选择能让用户最快看懂主结论的版式。
