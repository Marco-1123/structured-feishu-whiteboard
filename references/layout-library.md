# Layout Library

只从下面 5 个版式中选择。可以根据内容数量调整列数、卡片宽度、间距和高度，但不要改变版式的核心表达逻辑。

## 1. 结论先行

适用：

- 方案汇报、研究总结、战略建议、执行建议。
- 材料有明确主张或推荐路径。

结构：

- 顶部：主题标题 + 一句话主结论。
- 中部：3 到 5 个支撑模块卡片。
- 底部：关键行动、风险或下一步。

排版规则：

- 主结论占画板最强视觉位置。
- 主结论必须位于标题区下方或独立强视觉条中，不放在右上角小框里作为补丁。
- 支撑模块用等宽卡片，标题短、正文少。
- 行动项用窄条或底部栏，不抢主结论。
- 容量边界：1 个主结论、3 到 5 个支撑模块、1 条行动或风险提示。
- 支撑模块默认是并列关系，不使用箭头；只有明确时间推进或流程依赖时才加箭头。
- 卡片内部采用固定安全边距：标题、正文、标签之间保持稳定垂直间距，标签不能贴近卡片底边。
- 同一排卡片的内容基线要一致；不要让某张卡片文字过低、某张卡片文字过高。
- 连接箭头只放在模块之间的留白区，箭头中心线对齐相邻模块的视觉中线，不得压到正文或底部标签。

失败条件：

- 没有明确主结论。
- 内容主要是时间顺序或流程机制。

## 2. 问题拆解

适用：

- 诊断报告、现状分析、增长问题、组织问题、产品问题。
- 材料包含问题、原因、影响、解决方向。

结构：

- 左侧或顶部：核心问题。
- 中部：3 到 5 个原因或维度。
- 右侧或底部：对应策略、机会或治理动作。

排版规则：

- 使用“问题 -> 原因 -> 方案”的阅读方向。
- 原因模块之间保持同级，不混入行动项。
- 每个原因最好对应一个影响或证据。
- 容量边界：1 个核心问题、3 到 5 个原因、3 到 5 个治理动作。
- 原因卡片里只放问题和治理方向，不放完整指标区；指标多时拆为指标画板。
- 如果原因需要长证据或多层解释，先做总览，再拆证据画板。

失败条件：

- 材料不是诊断型，而是计划型或流程型。

## 3. 对比矩阵

适用：

- 竞品分析、方案比较、路径选择、能力评估。
- 材料包含多个对象和多个评价维度。

结构：

- 行：对象、方案、产品、阶段或角色。
- 列：评价维度。
- 右侧或底部：推荐结论。

排版规则：

- 维度控制在 3 到 5 个。
- 对象控制在 3 到 6 个。
- 单元格只放短语或符号化判断，不放长段落。
- 推荐项用强调色、边框或徽标突出。
- 容量边界：对象和维度的乘积建议不超过 25 个单元格。
- 如果单元格需要完整句解释，改为“对比总览 + 证据说明”两张画板。

失败条件：

- 只有单一对象，没有可比维度。
- 每个单元格都需要长解释。

## 4. 路线图 / 阶段规划

适用：

- 项目计划、产品路线、能力建设、实施路径。
- 材料天然按时间、阶段或成熟度展开。

结构：

- 横向阶段轴：3 到 5 个阶段。
- 每个阶段：目标、关键动作、交付物或指标。
- 末端：目标状态或成功标准。

排版规则：

- 阶段之间用箭头或连接线表达推进。
- 每阶段信息量保持均衡。
- 时间标签可以简短放在阶段标题附近。
- 容量边界：3 到 5 个阶段，每阶段 2 到 3 个要点。
- 阶段轴箭头应位于阶段标签之间的空白区，不能悬在说明文字上方或穿过说明文字。
- 如果阶段只是并列专项而非先后推进，不使用路线图箭头，改用结论先行或问题拆解。

失败条件：

- 没有顺序关系。
- 内容是多对象比较而非推进计划。

## 5. 流程 / 价值链

适用：

- 业务流程、用户旅程、系统链路、价值创造过程。
- 材料包含输入、处理、输出、角色或环节。

结构：

- 左到右主流程：4 到 7 个节点。
- 每个节点：动作、责任方、输入输出或关键风险。
- 上下辅助层：支撑能力、数据、约束或指标。

排版规则：

- 节点之间必须有明确方向。
- 连接线少而清晰，避免网状混乱。
- 支撑层用细卡片或横条，不打断主流程。
- 容量边界：4 到 7 个主节点，每个节点只保留 1 个动作和 1 个说明。
- 箭头表达的是“节点到节点”的关系，起点和终点必须靠近对应节点边缘或节点间空白，不要漂浮在画布中部。

失败条件：

- 材料主要是结论陈述，没有过程关系。
- 节点超过 7 个且不能合并。

## 选择优先级

1. 有明确推荐或判断：结论先行。
2. 有问题和原因：问题拆解。
3. 有对象和维度：对比矩阵。
4. 有阶段和时间：路线图。
5. 有输入输出和环节：流程/价值链。

如果多个版式都可用，选择能让用户最快看懂主结论的版式。
