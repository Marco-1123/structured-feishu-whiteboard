# Long-Form Workflow

长文不能直接画图。先把材料变成可承载的信息架构，再决定单图或多图。

## 1. 清理输入

先清理会破坏 SVG 文本的内容：

- 去掉无意义换行、重复标题、页眉页脚、目录编号残片。
- 把连续空格、制表符、项目符号混杂统一成短句。
- 保留中文、英文、数字和必要标点；删除异常控制字符。
- 将长 URL、脚注、来源路径降级为背景信息，不写进画板。
- 对 `<`、`>`、`&` 做文本转义，只保留 SVG/XML 标签本身。

## 2. 生成信息架构草稿

画图前必须先形成这个草稿：

```text
材料体量: 短 / 中 / 长 / 复杂
目标读者:
主结论:
版式选择:
风格选择:
单图或多图:
总览画板:
- 模块 1: 标题 / 2-3 个短句 / 可视化角色
- 模块 2: 标题 / 2-3 个短句 / 可视化角色
- 模块 3: 标题 / 2-3 个短句 / 可视化角色
降级信息:
需要拆出的细节画板:
```

这份草稿默认不输出给用户，但必须用于控制容量。

## 3. 长文压缩策略

按以下顺序压缩：

1. 找出材料真正要表达的 1 个判断或目标。
2. 把段落归并成 3 到 6 个模块。
3. 每个模块只保留 1 个短标题和 2 到 3 个短句。
4. 数据、案例、背景只保留能支撑主结论的部分。
5. 细节、过程、引用和长解释降级，不进入总览画板。

不要为了“覆盖原文”牺牲可读性。画板首先是结构化表达，不是全文摘要。

## 4. 单图 / 多图决策

- 用户只说“做成画板”“美化”“结构化展示”：默认做 1 张总览画板。
- 用户说“内容比较多也要都呈现”“做成系列”“分几张”：允许多画板。
- 内容同时包含问题、路线、对比和证据时：总览画板只放结论和目录式结构，细节拆图。
- 任一版式超过 `content-budget.md` 的容量上限时：先压缩；仍超载则拆图。

## 5. 生成顺序

1. 读取 `content-budget.md` 判断容量。
2. 读取 `layout-library.md` 选择主版式。
3. 读取 `style-library.md` 选择风格。
4. 先写总览 SVG。
5. 渲染检查，若超载按 `overflow-repair.md` 修复。
6. 需要多图时，再逐张生成细节画板。

## 6. 长文默认输出

长文默认输出应该说明：

- 使用了总览画板还是多画板。
- 哪些信息被合并或降级。
- 如果用户要完整展开，建议继续生成哪些细节画板。

不要把“因为材料很长所以我压缩了”等过程说明写在画板里；这些说明只放在回复中。
