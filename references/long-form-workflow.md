# Long-Form Workflow

长文不能直接画图。先把材料变成可承载的信息架构，再生成纵向大画布。总览是第一屏，不是完整输出。

## 1. 清理输入

先清理会破坏 SVG 文本的内容：

- 去掉无意义换行、重复标题、页眉页脚、目录编号残片。
- 把连续空格、制表符、项目符号混杂统一成短句。
- 保留中文、英文、数字和必要标点；删除异常控制字符。
- 将长 URL、脚注、来源路径降级为背景信息，不写进画板。
- 对 `<`、`>`、`&` 做文本转义，只保留 SVG/XML 标签本身。

## 2. 生成信息架构草稿

画图前必须先形成这个草稿：

```text
材料体量: 短 / 中 / 长 / 复杂
目标读者:
主结论:
输出形态: 纵向大画布 / 单屏总览
风格选择:

信息保全:
- 必须进入画板:
- 可以删除:

总览区:
- 模块 1: 标题 / 2-3 个短句 / 可视化角色
- 模块 2: 标题 / 2-3 个短句 / 可视化角色
- 模块 3: 标题 / 2-3 个短句 / 可视化角色

后续分区:
- 问题 / 背景:
- 关键模块展开:
- 路线 / 阶段:
- 指标与证据:
- 风险与待确认:
- 下一步行动:
```

这份草稿默认不输出给用户，但必须用于控制分区和容量。

## 3. 长文压缩策略

压缩只针对表达形式，不针对重要信息：

1. 找出材料真正要表达的 1 个判断或目标。
2. 把段落归并成 3 到 6 个核心模块。
3. 每个模块改写为短标题和短句。
4. 数据、案例、背景、约束、风险和行动进入对应分区。
5. 只删除重复话术、铺垫、格式残片、长 URL 和过程性描述。

不要为了“稳定”删除关键结论、约束、风险、指标、证据和行动。稳定性通过分区、增高画布和控制单屏密度实现。

## 4. 单屏 / 大画布决策

- 用户只说“做成画板”“美化”“结构化展示”：长文默认做 1 个纵向大画布。
- 用户明确说“只要总览”“一页就够”：才做单屏总览。
- 内容同时包含问题、路线、对比、证据、指标和风险时：第一屏放总览，后续分区完整展开。
- 任一分区超过 `content-budget.md` 的单屏容量上限时：新增下一个分区，不删除重要信息。

## 5. 生成顺序

1. 读取 `content-budget.md` 判断单屏和大画布容量。
2. 读取 `large-canvas-workflow.md` 形成信息保全清单和分区草稿。
3. 读取 `style-library.md` 选择风格。
4. 长文优先写 `layout: "large-canvas"` brief，再由脚本生成 SVG。
5. 渲染检查，若分区超载按 `overflow-repair.md` 修复。
6. 需要更多内容时，在同一纵向画布中增加分区。

## 6. 长文默认输出

长文默认输出应该说明：

- 使用了纵向大画布。
- 第一屏是总览，后续分区承载了哪些重要信息。
- 哪些内容被删除为低价值内容。

不要把“因为材料很长所以我压缩了”等过程说明写在画板里；这些说明只放在回复中。
