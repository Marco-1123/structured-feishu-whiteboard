# Report Workflow

本流程把材料转成“适合画板表达的信息结构”。不要先画图，先把内容压缩成可表达的结构。

## 1. 内容读取

接受任意来源，但读取职责分离：

- 飞书文档：使用飞书文档相关能力读取正文。
- 网页：使用浏览或抓取能力获取正文，不要只看标题。
- 文件：使用对应文档、PDF、表格或 Markdown 能力提取内容。
- 粘贴文本：直接处理。

读取后先判断材料类型：方案、报告、调研、竞品分析、规划、流程说明、会议纪要、知识总结或混合材料。

同时判断体量：

- 短内容：可直接进入结构化。
- 中等内容：先做总览画板，再决定是否需要细节。
- 长文 / 报告：必须读取 `content-budget.md` 和 `long-form-workflow.md`，先压缩，不直接画图。
- 混合材料：按最复杂的信息关系处理，不按字数低估难度。

## 2. 信息筛选

从材料中提取这些内容：

- 主题：一句话说明材料讨论什么。
- 主结论：最重要的判断、建议或方向。
- 关键模块：3 到 6 个支撑主结论的内容块。
- 证据：数据、事实、案例、用户反馈、约束条件。
- 关系：因果、并列、阶段、对比、层级、输入输出。
- 行动：下一步、责任、优先级、里程碑。

删除或降级：

- 重复表述。
- 背景铺垫中过长的叙述。
- 只服务原文语境、不服务画板理解的细节。
- 过程性说明和来源说明，除非用户明确要求展示。

## 3. 结构化输出草稿

生成 SVG 前，先在脑中形成这个结构：

```text
材料体量:
主题:
主结论:
目标读者:
单图或多图:
版式候选:
关键模块:
- 模块 1: 标题 / 1-2 句摘要 / 证据
- 模块 2: 标题 / 1-2 句摘要 / 证据
- 模块 3: 标题 / 1-2 句摘要 / 证据
可删除信息:
降级信息:
风险或不确定:
```

这份草稿不要完整输出给用户，除非用户要求。它用于指导画板生成。

如果任何模块的摘要超过 3 行，或关键模块超过 6 个，先回到信息筛选阶段继续压缩；不要进入 SVG。

## 4. 表达原则

- 画板标题应直接命名主题或结论，不写“某材料总结”。
- 每个模块标题用判断句或名词短语，不用长句。
- 正文以短句为主，每个卡片 1 到 3 行。
- 数字、阶段、对象、对比维度优先视觉化。
- 重要程度通过位置、字号、面积、颜色强调，而不是堆叠装饰。

## 5. 内容密度

默认单画板适合：

- 1 个主标题。
- 1 个主结论或导语。
- 3 到 6 个模块。
- 每个模块 1 到 3 条要点。
- 最多 1 个小型表格或矩阵。

超过这个密度时，先压缩，不要缩小字号硬塞。

## 6. 长文输出原则

- 长文默认产出“分屏画布”：第一张画框承载总览，后续画框承载细节、证据、风险、指标和行动。
- 总览画框只承载主结论、关键模块和推荐路径，不代表完整输出。
- 用户要求“完整呈现”时，在同一画板内横纵扩展多个画框，而不是扩大单张画框到不可读。
- 回复中可以说明哪些信息被合并或降级；画板上不要写制作过程。
